ADB使用技巧

发表于 2016-03-13   |   分类于 Android   |  

ADB

ADB，即 Android Debug Bridge，是 Android 开发中通过 PC 端控制 Android 设备的重要命令行工具。

ADB 是一个客户端-服务端程序，包含以下三部分功能：

  • 一个客户端，运行在开发 PC 上。可以通过 ADB 命令行来调用客户端。其他的 Android 工具，例如 ADT 和 DDMS 都是由 ADB 创建的。
  • 一个服务端，以后台进程运行在开发 PC 上。服务端用于客户端与模拟器或者 Android 设备上的守护进程的通讯。
  • 一个守护进程，以后台进程运行在模拟器或者 Android 设备上。

如何使用 ADB

  1. 配置 ADB 环境变量
    • Windows
      将 <Android SDK>/platform-tools/ 添加在系统环境到 path
    • Mac
      在配置文件中写入 export PATH=${PATH}:<Android SDK>/platform-tools/
  2. 打开手机中的 USB调试 功能。
  3. 将手机通过 USB 连接至 PC（可能需要安装驱动），在手机上同意授权。

无线连接

ADB还可以通过无线连接，仅需要简单配置即可实现在 Wi-Fi 下使用 ADB

  1. 将 PC 与手机连接至同一 Wi-Fi 下。

  2. 通过 USB 连接手机。

  3. 设置手机监听 TCP/IP 5555 端口

    若PC连接多个手机，需要指定设备

  4. 查询手机的内网 ip 地址。

  5. 连接手机

  6. 确定是否已连接

录制视频

在 Android 4.4（API 19）之后的设备上，可以通过 screenrecord 命令方便地录制视频。录制的视频格式为 MPEG-4 。

注意：录制的视频只有画面，没有声音。

部分设备不支持原生屏幕分辨率，尝试低分辨录制
录制过程中不支持屏幕旋转，若发生旋转，则部分画面会被裁剪。

常用命令

查看已连接设备

查看已连接的设备，包括 USB 连接与 TCP/IP 5555 端口连接的设备。

  • Serial number
    • 真实设备显示为控制台序号
    • 通过 TCP/IP 5555 端口连接的会显示 192.168.xxx.xxx:5555
    • 模拟器会显示为 emulator-xxxx
  • state
    • offline - 已连接但设备无响应。
    • device - 已连接至adb服务。adb服务在设备开机时即开始加载，故设备可能处于开机状态。
    • no device - 没有设备或模拟器连接。

添加 -l 可查看设备限定词

远程shell

运行手机内的 /system/bin/... 的命令

  • 进入连接设备的shell命令行

    输入 exit 退出远程shell或者按 CTRL+D 。

  • 单次使用远程shell命令

选择设备

当只连接单个设备或者模拟器时，adb shell会默认选择已连接的设备。当连接多个设备时，有些命令需要指定选择的设备。可通过 -d|-e|-s <specific device> 选择设备。

-d - 选择通过USB连接的设备，当有多个设备通过USB连接时返回错误。
-e - 选择模拟器，当有多个模拟器时返回错误。
-s <specific device> - 根据设备序列号或者限定词选择设备。

Java核心技术-基础知识 学习笔记01

发表于 2016-03-11   |   分类于 学习笔记 ， Java   |  

序

开发 Android 近一年，但始终没有静下心来好好巩固下 Java 的基础。
目前看的《Java核心技术 卷I 基础知识》第九版，可以说是一本经典的 Java 基础书籍，值得推荐。打算把我的学习笔记发到博客，以此来激励自己(。・＿・。)ﾉ

Java概述

Java 不只是一种语言，是一个完整的平台，有一个庞大的库，提供了注入安全性、可移植性强以及自动垃圾收集等执行环境。

Java”白皮书”中的关键术语

简单性

Java 语法是 C++ 语法的一个“纯净”版本。

说到 Java，无疑会和 C++ 对比。Java 剔除了 C++ 中很少使用的、又难以理解的特性：头文件、指针运算、结构、联合、操作符重载、虚基类等。
另外，Java 运行的环境所需很小，基本解释器大约仅为 40KB；加上标准类库和对线程的支持大约增加至 175KB。现在Java对嵌入式设备提供了 Java Micro Edition 微型版运行环境。

面向对象

OOP 已经证明了是现代编程的主流方向，Java 提供了类、接口等面向对象技术。

网络技能

Javat 提供了用于 HTTP 和 FTP 之类的 TCP/IP 协议的例程库，使用 Java 开发 socket 网络协议变得简单。

健壮性

Java 在设计之初就投入了大量精力考虑程序的可靠性，Java 的指针模型相对 C++ 消除重写内存和损坏数据的可能性。Java 编译器能够检测许多其他语言仅在运行时刻才能检测出来的问题。

安全性

在 Java 发展中也不断地添加更多安全特性，Java 最初便了防范各种共计：

  • 运行时对栈溢出。
  • 在自己的处理空间之外破坏内存。
  • 未经授权读取文件。

体系结构中立

Java 编译器通过生成与特性计算机体系结构无关的字节码指令来实现这一特性。

虽然解释字节码比全速运行机器码指令慢很多，但 Java 将使用最频繁的字节码序列翻译成机器码，这一过程被称为即时编译。这一策略已经被证明十分有效，.NET 也采用了这种策略。

可移植性

Java 规范中没用“依赖具体实现”的地方。基本数据类型的大小以及有关算法都做出了明确的说明。

例如：Java 中 int 永远为 32 位整数，不会因为操作系统或编译器的不同产生变化。Java 中数据类型具有固定的大小，二进制数据以固定的格式进行存储和传输。字符串使用标准的Unicode格式存储的。

解释型

Java 解释器可以在任何环境中在中执行 Java 字节码。现在即时编译器将字节码翻译成该环境下的机器码。

高性能

在一些场合下，为了更高性能，在运行中字节码可以快速地翻译成该环境下的 CPU 机器码。这使得即时编译器成为了传统编译器的竞争对手，有些情况下甚至能优于传统编译器。

多线程

Java 把多线程的实现交给了底层的操作系统或者线程库来完成。Java 在多线程开发上的便捷性是值的肯定的。正因为如此， Java 成为了诸多服务器的开发语言。

动态性

Java 库可以自由地添加新方法和实例变量，而对客户端却没有影响。

Java 当前版本允许程序员了解队形的结构和行为，这对于必须在运行时分析对象的系统来说至关重要。
